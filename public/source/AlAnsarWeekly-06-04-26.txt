Publication identifiers are not always numeric.

Examples:

```text
8-9
10-11
vol 59
Special Issue
```

System previously assumed:

```js
Number(samputa)
Number(sanchike)
```

Result:

```text
8-9 -> NaN -> null
```

Metadata was lost.

---

### Frontend Fixes

Changed inputs from:

```jsx
type="number"
```

to:

```jsx
type="text"
```

in:

* SingleUpload
* ManageQnA modal

Also removed:

```js
Number(e.target.value)
```

from edit handlers.

---

### Backend Fixes

Found issue inside upload route.

Old code:

```js
const normalizedSamputa =
  samputa !== null &&
  samputa !== "" &&
  !isNaN(Number(samputa))
    ? Number(samputa)
    : null;
```

and similar for sanchike.

Replaced with:

```js
const normalizedSamputa =
  samputa && samputa.trim() !== ""
    ? samputa.trim()
    : null;

const normalizedSanchike =
  sanchike && sanchike.trim() !== ""
    ? sanchike.trim()
    : null;
```

without future schema changes.
